# h1 元素的預設樣式即將改變

> 原文發佈日期：2025-04-11,作者 Simon Pieters。

## TL;DR

各家瀏覽器即將移除 `<h1>` 在巢狀 sectioning 元素 (`<section>`、`<article>`、`<nav>`、`<aside>`) 內的特殊預設樣式。若你的網站「依賴」這個隱式行為去產生視覺層級,改版後 `<h1>` 就會全部呈現同樣大小,Lighthouse 也會跳警告 `H1UserAgentFontSizeInSection`。

**因應做法:不要依賴 UA 預設樣式表達層級。請用 `<h2>`/`<h3>` 表達層級,並為 `<h1>` 明確指定 `font-size` 與 `margin`。**

## 背景:被廢棄的 outline algorithm

過去 HTML spec 定義了一個 outline algorithm,讓 `<h1>` 的「隱式層級」依被多少個 sectioning 元素包覆而改變。瀏覽器在 UA stylesheet 裡實作對應的視覺降級,例如:

```css
/* x is :is(article, aside, nav, section) */
x h1 { margin-block: 0.83em; font-size: 1.50em; }
x x h1 { margin-block: 1.00em; font-size: 1.17em; }
x x x h1 { margin-block: 1.33em; font-size: 1.00em; }
x x x x h1 { margin-block: 1.67em; font-size: 0.83em; }
x x x x x h1 { margin-block: 2.33em; font-size: 0.67em; }
```

但這個 outline algorithm:
- **沒有寫進 accessibility tree** (螢幕閱讀器讀到的還是 `<h1>`),只影響視覺。
- 開發者對「何時可以用 `<h1>`」混亂,各家工具處理不一致。
- 已於 2022 年從 HTML spec 移除,但 UA stylesheet 裡對應的規則還留著。瀏覽器現在要把這部份也拿掉。

範例 HTML:

```html
<body>
  <h1>Level 1</h1>
  <section>
    <h1>Level 2</h1>
    <section>
      <h1>Level 3</h1>
      <section>
        <h1>Level 4</h1>
      </section>
    </section>
  </section>
</body>
```

舊行為:Level 2~4 視覺上漸縮。新行為:四個 `<h1>` 視覺一樣大。

## 各瀏覽器時程

### Firefox
- 2025-03-31 起,Firefox Beta 138 對 50% 使用者套用新行為。
- 4 月 14 日起 Beta 138 100% 套用。
- 穩定版 138 由 5% → 10% 逐步放量,Firefox 140 全平台上線。對應 bug:`1885509`。
- 自 Firefox 136 起,DevTools console 會對未指定 `font-size`/`margin` 的 `<h1>` 顯示警告 (bug `1937568`)。
- 可在 `about:config` 設 `layout.css.h1-in-section-ua-styles.enabled` 為 `false` 提前測試。

### Chrome
- 自 Chrome 136,當 `<h1>` 套用 UA 預設較小字級時會顯示 deprecation 警告。在 Chromium 標記 deprecated 會降低 Lighthouse 的 "Best Practices" 分數 (issue `394111284`)。

### Safari
- 追蹤 bug `292765`。

## 修法

當 Lighthouse 報 `H1UserAgentFontSizeInSection` 時,為 `<h1>` 加上明確的 `font-size` 與 `margin`:

```css
h1 {
  margin-block: 0.67em;
  font-size: 2em;
}
```

若不想覆蓋其他針對 `<h1>` 的規則,可以用 `:where()` (零 specificity):

```css
:where(h1) {
  margin-block: 0.67em;
  font-size: 2em;
}
```

## 行動清單

- 不要依賴瀏覽器預設樣式表達標題層級;用 `<h2>`/`<h3>` 等。
- 為 `<h1>` 明確定義 `font-size` 與 `margin`。
- 更新 CSS reset 以反映新行為。
- 用 Lighthouse 與 DevTools 稽核站點。
